## 프론트엔드, 디자이너 포지션과 협업
### API 명세
- 알림 클릭 시, 관련 페이지로 이동하기 위한 API 응답 명세 논의
	- 다양한 의견이 합쳐질수록 더 좋은 아이디어가 도출되고, 협업의 즐거움을 느낌

- 프론트엔드 팀원과 적극적인 의사소통, 지속적 갱신
	- 프론트엔드에서 작업하기 쉬운 응답 명세를 위해 노력함
	- 이 과정에서 DTO 매핑 로직이 다소 복잡해지는 트레이드 오프를 느낌
	  

### QA 과정에서 프론트엔드와 트러블슈팅
- 로그인 시도 무한 루프 문제
	- 아쉬웠던 점: 프론트엔드의 로직 파악이 쉽지 않았음
	- 개선해야할 점: 현재 내가 처한 상황을 전달하고, 문제의 핵심을 파악하기 위해 지속적 노력해야 함. 마찬가지로 백엔드의 로직을 이해하기 쉽게 전달해야 함

	- 트러블슈팅이 어려웠던 점은 백엔드의 Custom Exception 계층 구조에 대한 컨벤션이 명확하지 않았기 때문임. 이에 대한 재발 방지 대책 수립 필요
	- 또한 인증 로직에 대한 이해가 아쉬웠음.
	- 또한 인증 필터 단에서 핸들링해야 할 인증 관련 Exception들을 글로벌 예외 핸들링에서도 핸들링하고 있다보니, 401 에러가 발생했을 때 확인해야 할 범위가 넓어졌음
